2. The ADT binary tree

The class LinkedBinaryTree. The class LinkedBinaryTree is a node-based implementation of the ADT binary tree. It is very similar to the class LinkedBinaryTree given in §7.3.4, pp. 308–312, of our main textbook. The only difference with the textbook version is that this class uses ArrayIndexList objects whenever the textbook version uses NodePositionList objects (in particular, to implement iterators). In the class LinkedBinaryTree, most of the methods are already implemented. You are to implement the following two methods according to the given specifications.
  1. preorderElements()—Returns an Iterable collection of all the elements in preorder.
  2. postorderElements()—Returns an Iterable collection of all the elements in postorder.
For each of these methods, you should create and return an ArrayIndexList object as an Iterable collection. You may also add an auxiliary method to perform the preorder/postorder traversal recursively. A test driver program is provided for you in the file Tester.java. Run this driver to test the correctness of your implementation. Save a snapshot your test run in a file named lab8.out.

3. Expression trees

The class ExpressionTree. Next, open the file ExpressionTree.java. The class ExpressionTree is a subclass of the class LinkedBinaryTree, and it is designed to represent expression trees of arithmetic expressions (see also Example 7.9, p. 301, of our main textbook). In an expression tree, each internal node holds one of the arithmetic operators +, -, * and /, and each external node holds integers. We assume that both operators and digits are represented in String objects. In the class ExpressionTree, implement the following two methods according to the given specifications.
  1. evaluate()—Returns the integer value of the entire expression tree.
  2. evaluate(v)—Recursively evaluates and returns the integer value of the expression tree rooted at v.
To convert a string, say, s, consisting of digits to its corresponding integer value, use the class Integer's constructor: 
new Integer(s)
To test equality of two strings, say, s and t, use the class String's method: 
s.equals(t)
Testing correctness. In the prefix notation, arithmetic operators appear before operands. For example, addition of 1 and 2 in the standard (infix) notation is 1 + 2, but in the prefix notation it is + 1 2. It turns out that the preorder traversal of an expression tree yields its expression in the prefix notation. In the file Tester.java, test your implementation with the following arithmetic expressions in the prefix notation as input: 
- / * + 3 1 3 + - 9 5 2 + * 3 - 7 4 6
+ - 496 * 28 6 8128
The method preorderBuild() is provided for you to construct an expression tree from an arithmetic expression in the prefix notation. Save a snapshot your test run in the file lab8.out.
