2. The ADT dictionary

The ADT dictionary is defined by the following seven methods. In the ADT map, all entries must have distinct keys; however, in this ADT dictionary, different entries may share the same key.
  1. size()—Returns the number of entries in the dictionary.
  2. isEmpty()—Returns whether the dictionary is empty.
  3. get(k)—Returns an entry whose key is equal to a given key k.
  4. getAll(k)—Returns an iterable collection of all entries whose keys are equal to a given key k.
  5. put(k, x)—Puts an entry with a given key k and value x into the dictionary, returning the entry created.
  6. remove(e)—Removes from the dictionary a given entry e, returning the removed entry.
  7. entrySet()—Returns an iterable collection of all key-value entries in the dictionary.
The class HashTableDictionary implements the ADT dictionary using a hash table, where linear probing is used for collision handling. Except for the two methods getAll() and entrySet(), all of the above methods are implemented. Study carefully how this class is designed. A hash function is defined in the method hashValue() using the simple division method (see also p. 396 of our main textbook).

Getting all entries. To complete the implementation of the ADT dictionary, complete the methods getAll() and entrySet() according to the specifications given above. As usual, for iterable collections, use the class ArrayIndexList. In the method getAll(), you should begin your search from the index returned by the auxiliary method findEntry(). A partially completed test driver program is provided for you in the file Tester.java. Using iterators, add code to Tester.java to test your implementation.

Removing all entries. Now, in the class HashTableDictionary, implement the following additional method according to the given specification.

  1. removeAll(k)—Removes all entries whose keys are equal to a given key k and returns the removed entries in an iterable collection.
You should begin your search from the index given by the auxiliary method findEntry(). Using iterators, add code to Tester.java to test your implementation. Once completed, save a snapshot of your test run in a text file named lab10.out.
